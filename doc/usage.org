#+TITLE: Build & Deploy Quick-Start
#+OPTIONS: toc:2 num:nil

* Quick-start

All commands are run from the repository root unless otherwise noted.

** Prerequisites

- Guix must be on =$PATH=.
- This repo must be accessible at a path the =guix= command can load modules from.
- Use the =-L .= flag to add the repo root to the Guix load path.

** Real machine — home environment only

Reconfigure the home environment for the current user:

#+BEGIN_SRC sh
guix home reconfigure -L . systems/desktop.scm
#+END_SRC

To use the exact pinned Guix commit from =channels.scm=:

#+BEGIN_SRC sh
guix time-machine --channels=channels.scm -- \
  home reconfigure -L . systems/desktop.scm
#+END_SRC

** VM — full OS (ephemeral, fast iteration)

Build a =qemu-kvm= launch script and run it:

#+BEGIN_SRC sh
vm=$(guix time-machine --channels=channels.scm -- \
       system vm -L . systems/vm.scm)

$vm -- \
  -enable-kvm -m 2048 -smp 2 \
  -device virtio-gpu \
  -display gtk,gl=off \
  -nic user,model=virtio-net-pci
#+END_SRC

Share the repo into the VM for live editing:

#+BEGIN_SRC sh
$vm --share=$PWD=/home/ivand/config -- \
  -enable-kvm -m 2048 -device virtio-gpu -display gtk,gl=off
#+END_SRC

** VM — persistent qcow2 image

#+BEGIN_SRC sh
img=$(guix time-machine --channels=channels.scm -- \
        system image -L . \
        --image-type=qcow2 --image-size=40GiB \
        systems/vm.scm)

cp -L "$img" /tmp/guix-desktop.qcow2
chmod +w /tmp/guix-desktop.qcow2

qemu-system-x86_64 \
  -enable-kvm -m 4096 -smp 4 \
  -drive file=/tmp/guix-desktop.qcow2,if=virtio \
  -device virtio-gpu \
  -display gtk \
  -nic user,model=virtio-net-pci
#+END_SRC

** VM login

| Field | Value |
|-------|-------|
| User | =ivand= |
| Password | (empty — press Enter) |

Sway starts automatically on =tty1= via =.bash_profile=.

** Module syntax check

To quickly verify that =systems/common.scm= loads without errors (without
actually building anything):

#+BEGIN_SRC sh
guix build --no-grafts \
  -e '(begin (add-to-load-path ".") (use-modules (systems common)) "ok")' \
  -L . 2>&1 | grep -E "error|warning|ok"
#+END_SRC

A clean load produces only:
#+BEGIN_EXAMPLE
guix build: error: "ok": not something we can build
#+END_EXAMPLE
(The module loaded fine — "ok" just isn't a buildable Guix object.)

** Wallpaper setup

The wpaperd service expects a wallpaper directory at =~/Pictures/bg= by
default.  Create it and populate it before first login:

#+BEGIN_SRC sh
mkdir -p ~/Pictures/bg
cp /path/to/wallpaper.jpg ~/Pictures/bg/
#+END_SRC

To change the path or rotation duration, override the service in
=systems/common.scm= or in a machine-specific config.  See
[[file:services.org::*home-wpaperd-service-type][services.org]] for details.

** Updating pinned channels

Edit =channels.scm= and update the =:commit= field to the desired Guix commit:

#+BEGIN_SRC scheme
(list (channel
       (name 'guix)
       (url "https://codeberg.org/guix/guix.git")
       (commit "NEWCOMMIT")))
#+END_SRC

Then re-run your =guix time-machine= commands.
